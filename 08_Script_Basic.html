자바스크립트 개요


1. 스크립트 언어란?

프로그램의 동작을 사용자의 요구에 맞게 수행하도록 해주는 용도

매우 빠르게 배우고 짧은 소스코드로 상호작용되도록 고안됨

2. 자바스크립트

웹 브라우저 많이 사용되는 "인터프리터 방식"의 스크립트 언어

- 인터프리터 방식 : 코드를 한줄 한줄 읽어가며 바로 실행하는 방식

(실시간으로 텍스트를 분석해서 실행됨)

=> 코드에 문법오류가 있으면 실행시점에 알려줌

3. 장단점

장점

  • 컴파일 과정 없이 인터프리터로 코드를 한 줄씩 읽어가며 바로 실행 (수행속도 빠름)
  • 코드 작성이 간단한 편. 단순구조. ex) 접근제한자 개념없음, 변수 선언 시 자료형을 정할 필요 없음

단점

  • 웹에 특화된 기술 => 내부에서 제공되는 기능이 제한적
  • html 문서 내에 소스코드를 작성하면 외부에 공개가 된다. => 보안상 취약함.

4. 자바스크립트 작성방식

  1. inline (인라인) : 태그에 속성으로 직접 작성하는 방식 (간단한 소스코드)
  2. internal (내부) : html 문서 내에 script 태그를 사용하여 작성하는 방식
  3. external (외부) : 별도의 .js 파일을 만들어 작성하는 방식

1) inline(인라인)

태그 내에 간단한 소스코드를 작성하는 방식

[표현법] <태그명 on이벤트명="해당 요소에 해당 이벤트 발생 시 실행할 코드" >


2) internal(내부)

html 문서 내에 script 태그를 사용하여 영역을 만들고 영역 안에 소스코드를 작성

작성 위치는 어디든 가능. 되도록 head, body태그 안에 작성하도록 하자.


3) external(외부)

별도의 .js 파일로 소스코드를 작성하여 html문서에 연결하는 방식

* 접근한 요소의 값을 가지고 오는 방법
