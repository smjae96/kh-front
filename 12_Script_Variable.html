변수와 자료형

변수 선언 시 var, let, const를 이용하여 변수를 선언

* var, let, const의 차이점

1) 변수 선언 방법

* 자료형

변수 선언 시 자료형을 별도로 지정하지 않음 (자료형의 개념이 없는 건 아님!)
=> 변수에 대입되는 값(리터럴)에 따라서 알아서 자료형이 결정됨

자바스크립트의 자료형

  • String(문자열)
  • number(숫자)
  • boolean(논리값)
  • object(객체)
  • function(함수)
  • undefined(초기화가 되지 않은 변수)

* 변수 명명 규칙

  1. 일반적으로 영어를 사용하며, 문자와 숫자 모두 사용 가능
  2. 특수문자 $ 또는 _만 사용 가능
  3. 숫자로 시작할 수 없음 ex) 1004aa
  4. 키워드를 변수명으로 사용할 수 없음 ex) let, var, .. 등

변수명 표기법 (Naming Type)

  1. camelCase => 대부분 언어에서 사용되는 표기법. 단어의 시작마다 대문자로 구분
  2. snake_case => 파이썬에서 주로 사용되는 표기법. 단어의 시작마다 _로 구분
  3. Pascalcase => c# 또는 ms계열의 언어에서 사용되는 표기법. 모든 단어의 시작을 대문자로 표기

(참고) Hoisting ?

모든 변수 선언문이 코드의 최상단으로 이동되는 것처럼 느껴지는 현상
var 타입에서 확인이 되고, let과 const는 호이스팅(hoisting)이 일어나지 않는 것처럼 동작하게 해줌.
